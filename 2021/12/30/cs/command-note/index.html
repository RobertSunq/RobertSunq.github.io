配置SSH免密登录

# 生成 SSH 密钥对, 将生成一对公钥和私钥文件，默认情况下，它们将保存在 `~/.ssh` 目录下
[root@qing centos]$ ssh-keygen -t rsa -b 4096


# 复制公钥到目标服务器
# ssh-copy-id user@server_ip
[root@qing centos]$ ssh-copy-id -i robert@192.168.1.112
# 这将自动将公钥添加到目标服务器的 `~/.ssh/authorized_keys` 文件中，并为你进行一次免密登录验证。
# 如果 `ssh-copy-id` 命令不可用，或者你使用的是 Windows 操作系统，可以手动将公钥内容复制到目标服务器的 `~/.ssh/authorized_keys` 文件中。

# 如果 ssh-copy-id 无法使用，也可用手动将文件上传到指定的服务器上
## $ scp /home/robert/.ssh/id_rsa.pub robert@192.168.1.112:/home/robert/.ssh/authorized_keys_tmp

# 之后到192.168.1.112 将 authorized_keys_tmp 追加到文件 authorized_keys 中
## $ cat /home/robert/.ssh/authorized_keys_tmp >> /home/robert/.ssh/authorized_keys


# 测试免密登录
# ssh user@server_ip
[root@qing centos]$ ssh robert@192.168.1.112
# 如果一切正常，你将能够直接登录到目标服务器而无需输入密码

maven-surefire单元测试

部分命令对版本有所要求，请注意。

# 根据配置文件，运行范围内的所有单元测试
[root@qing centos]$ mvn test

# 指定某个包下面的所有用例
# [root@qing centos]$ mvn test -Dtest=${包完整路径}.*
# 最后的 .* 是通配符表示运行该包下所有用例
[root@qing centos]$ mvn test -Dtest=cn.qing.demo.*

# 指定某个测试用例类
# [root@qing centos]$ mvn test -Dtest=${包完整路径}.${测试用例类名}
[root@qing centos]$ mvn test -Dtest=cn.qing.demo.DemoTest

# 指定运行多个测试用例类
# [root@qing centos]$ mvn test -Dtest=${测试用例类名1},${测试用例类名2}
[root@qing centos]$ mvn test -Dtest=cn.qing.demo.DemoTest1, cn.qing.demo.DemoTest2

# 指定运行测试用例类中的一个方法
# [root@qing centos]$ mvn test -Dtest=${类路径}#${方法名}
[root@qing centos]$ mvn test -Dtest=cn.qing.demo.DemoTest#demoTest

# 以上方法可使用 * 进行通配
# mvn test -Dtest=cn.qing.demo.DemoTest#demo_*
# 上面的命令意思为 运行DemoTest测试类中的以demo_为开头的方法。

windows的默认编码为GBK，Linux的默认编码为UTF-8

因为在windows下编辑的中文，在linux下会显示为乱码。

复制文件

常规使用

# [root@qing centos]$ cp [option] ${source_file_path} ${target_file_path}
[root@qing centos]$ cp /tmp/qing/source.log /tmp/qing/source.log
-a archive 复制所有的目录
# -d 源文件为连接文件时，则复制连接文件属性而非文件本身
# -f 强制复制，如果遇到重复或者其他疑问，不会进行询问而强制复制
# -i 若目标文件已存在，在覆盖时先询问是否真的操作
# -I 建立硬连接的连接文件，而非复制文件本身
# -p 与文件的属性一起复制，而非使用默认属性
# -r 递归复制，用于目录的复制操作
# -s 复制成符号连接文件，“快捷方式”
# -u 若目标文件必源文件旧，则更新目标文件

强制复制

# 复制时，如果遇到重复或者其他疑问，不会进行询问而强制复制
[root@qing centos]$ cp -f /tmp/qing/source.log /tmp/qing/source.log

# 上述命令如果还是会询问，则可能是cp命令属于 其他命令组合的别名
# 使用alias 查看
[root@qing centos]$ alias
alias cp='cp -i'
# 此配置即为还会进行提示

# -f的替换方案
## 使用命令 \cp    反斜杠来强制复制
[root@qing centos]$ \cp /tmp/qing/source.log /tmp/qing/source.log

修改环境变量

修改环境变量文件

一些常见的环境变量相关的文件地址

  1. 全局环境变量文件

    /etc/profile 或 /etc/environment这些文件包含了全局的环境变量设置，对所有用户都生效。

  2. 用户特定环境变量文件

    ~/.bash_profile、~/.bashrc 或 ~/.profile这些文件位于用户的主目录下，用于配置用户特定的环境变量。不同的shell使用不同的文件

  3. 当前会话的环境变量

    在当前会话中设置的环境变量只在该会话中有效，会话结束后就会失效。可以使用export命令临时设置环境变量，例如

    export MY_VARIABLE="custom value"
export PATH="$PATH:/custom/path"

如果你需要查看当前环境变量的值，可以使用printenv命令或echo $VARIABLE_NAME来显示特定环境变量的值。

以JAVA的环境变量为例修改， 

# 当然也可以修改其他配置文件
[robert@cluster01 conf]$ sudo vim /etc/profile 
################## 写入以下内容 ################## 

### mysql config start ###
JAVA_HOME_11=/usr/local/java/jdk-11.0.17
JAVA_HOME=${JAVA_HOME_11}
CLASSPATH=${JAVA_HOME}/lib:${JAVA_HOME}/jre/lib
PATH=$PATH:${JAVA_HOME}/bin:${JAVA_HOME}/jre/bin
export CLASSPATH JAVA_HOME
### mysql config end ###

export PATH
################## 写入以上内容 ################## 


# 使配置文件生效 source /etc/profile
[robert@cluster01 conf]$ source /etc/profile

在全局shell脚本文件声明

/etc/profile.d/目录用于存放系统范围的shell脚本文件，这些脚本在用户登录时会自动执行。它们用于设置全局的环境变量、路径和其他系统级别的配置。

当用户登录时，登录shell会读取/etc/profile文件，并检查/etc/profile.d/目录是否存在。如果该目录存在，登录shell会按照一定顺序执行该目录下的所有脚本文件，以完成系统范围的环境设置。

通常，每个脚本文件都是独立的，以.sh为文件扩展名，并包含了特定的环境变量或系统配置。这种结构的好处是，系统管理员可以轻松地添加、修改或删除特定的配置文件，而无需修改主要的/etc/profile文件。

以JAVA的环境变量为例修改， 

# 编辑一个脚本文件
[robert@cluster01 conf]$ vim /etc/profile.d/env_robert.sh
################## 写入以下内容 ################## 

#!/usr/bin/env bash

export MY_VARIABLE="custom value"
export PATH="$PATH:/custom/path"

################## 写入以上内容 ##################


# 使配置文件生效 source /etc/profile
[robert@cluster01 conf]$ source /etc/profile

# 在用户登录时，这些导出的环境变量将被设置，并且/custom/path将被添加到$PATH中。

防火墙设置

# 永久开启暴露端口
$ firewall-cmd --zone=public --add-port=2181/tcp --permanent
# or
$ firewall-cmd --zone=docker --add-port=13306/tcp --permanent
# --zone 作用域
# --add-port=80/tcp  添加端口，格式为：端口/通讯协议
# --permanent  永久生效，没有此参数重启后失效


# 重启防火墙
$ firewall-cmd --reload
# or
$ systemctl restart firewalld.service


# 查看端口是否暴露
$ firewall-cmd --zone=public --query-port=8080/tcp


# 启动防火墙
$ systemctl start firewalld.service


# 设置开机自启
$ systemctl enable firewalld.service


# 检查防火墙状态是否打开
$ firewall-cmd --state


# 关闭防火墙
$ systemctl disable firewalld


# 检查防火墙开机启动是否生效
$ systemctl is-enabled firewalld.service;echo $?


# 查看防火墙信息
$ firewall-cmd --list-all
