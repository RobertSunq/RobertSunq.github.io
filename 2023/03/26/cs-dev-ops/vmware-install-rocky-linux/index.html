基于VMware安装Rocky linux

Rocky linux 是一个开源、社区拥有和管理、免费的企业级Linux发行版，它是由 CentOS 创始人 Gregory Kurtzer 发起的一个社区项目，旨在提供一个免费的、开源的、稳定的、安全的替代品，以取代Centos停止维护（改为滚动更新的Stream版）后，RHEL的下游Linux操作系统的替代方案，并继承了原CentOS的开源免费特点。

官方网址：https://rockylinux.org/

中文网址：https://www.rockylinux.cn/

阿里云镜像站：https://mirrors.aliyun.com/rockylinux/

本文基于 VMware Workstation 17 Player 安装，不同版本的VM有不同的配置顺序，但是大致模块和带配置内容都一致，所以按照自己实际情况参考进行配置安装。

为虚拟机配置网络连接

推荐选择桥接网络，这个也可以在后期进行自定义的修改，也可以配置多网络连接。当然如果没有自定义IP的需求直接选择 NAT 模式最为简单。

  • 桥接网络

    • 简单来说虚拟机相当于在宿主机所在局域网内的一个单独的主机，他的行为和宿主机是同等地位的，没有依存关系。该模式下虚拟机和宿主机地位均等，通过各自独立的IP链接网络，相当于虚拟机直接连接网络。通过网络连接中的 Vmnet0 与外界联系。

  • 网络地址转换（NET）

    • 简单来说虚拟系统借助NAT(网络地址转换)功能，通过宿主机所在的网络来访问公网，依赖于宿主机，如果是在公司内网中，推荐使用该模式，不需要再申请内网中的独立IP便可以接入网络。通过网络连接中的 Vmnet8 与外界联系。

  • 仅主机模式网络

    • 简单来说，就是把虚拟机与公网进行了隔离，建立了一个与其他机器相隔离的系统不可以访问外网，仅可以与宿主机进行通信，相当于将虚拟机与宿主机通过双绞线进行了链接。通过网络连接中的 Vmnet1 与外界联系

具体可参考如下两篇文章中的解析：

https://blog.csdn.net/pursuing0my0dream/article/details/50286481

https://blog.csdn.net/weixin_34418883/article/details/92515092

这里的三种网络链接，可以查看自己的网络适配器，其中会有两个VMware的网络配置，

  • VMware Network Adapter VMnet1

    • 宿主机用于与Host-Only虚拟网络进行通信的宿主机使用的虚拟网卡；即用于仅主机模式类型的网络。

  • VMware Network Adapter VMnet8

    • 宿主机用于与NAT虚拟网络进行通信的宿主机使用的虚拟网卡；即用于网络地址转换类型的网络。

同时在VMware的编辑中还有虚拟网络编辑器选项，在其中可以看到这两个网络配置进行修改，VMnet1，VMnet8，在点击更改设置的时候还可以看到VMnet0(是用于桥接网络)，剩下的VMnet2- VMnet7可以进行自定义custom网络下的虚拟交换机。

基础步骤

从上至下分别是：

  • 正常安装 Rocky Linux 9.1
  • 测试此光盘后再进入 Rocky Linux 9.1 安装步骤
  • 进入除错模式。选择此模式会出现更多的选项，分别是：

    • 由本地磁盘正常启动，不由光盘启动。

基本上，除非硬件系统有问题，包括有用比较特别的显卡外，否则使用“正常安装 Rocky Linux 9.1”的步骤即可。如果怀疑光盘由问题，就可以选择测试光盘后在进入Rocky Linux 9.1 安装的程序。如果确信此光盘没问题，就不要测试了。如果不在乎花费一两分钟的时间去测试看看光盘有没有问题，就使用测试后安装的步骤。

可以直接选中第一个选项 Install Rocky Linux 9.1来安装系统

额外配置

该步骤并不是必要步骤，只是为了强制使用 GPT 分区表的安装参数。

因为，如果磁盘容量小于2TB的话，系统默认会使用MBR分区表来安装。因为我们配置的默认磁盘容量是 40GB，所以默认肯定会用MBR分区表来安装。那如果想要强制使用GPT分区表的话，就需要使用以下步骤：

  1. 使用箭头键，将图image_12的光标移动到 Install Rocky Linux 9.1 选项

  2. 按下键盘的 Tab 按键，让光标移动到界面最下方输入额外的内核参数

  3. 再出现的界面中，输入以下参数

    vmlinuz initrd=initrd.img inst.stage2=hd:LABEL=Rocky-9-1-x86_64-dvd quiet inst.gpt

    其中按下tab键后会自动出现如图 image_14 所示的基本参数，在后面添加上 inst.gpt 即可。

    这里的重点参数就是输入的 “inst.gpt”。之后回车运行。

参数基本格式解析如下“

inst.ks=hd:LABEL=<LABEL NAME>:<KS FILENAME> inst.gpt

其中，<LABEL NAME> 是安装介质的标签名称，<KS FILENAME> 是包含 Kickstart 文件的文件名。使用此内核参数将强制在安装过程中使用 GPT 分区表。

请注意，在使用此内核参数之前，请确保系统支持 GPT 分区表。如果系统使用 BIOS 引导，则需要使用 MBR 分区表。如果系统使用 UEFI 引导，则可以使用 GPT 分区表。

此外，如果使用 UEFI 引导，还需要在启动时选择 UEFI 模式。如果使用 BIOS 引导，则应选择 Legacy 模式。

启动安装后

输入合适参数，启动安装之后，系统会运行一段检测的画面，这段检测的流程依据光驱速度、硬件复杂度而有不同，等待个几秒种到一两分钟就可以了。

下面先来制作第一个GPT 分区表最好要有的 BIOS boot 分区，如下图

先点选 1 号处的选项，不要使用默认的 LVM。选择 标准分区 选项，并按下 2 号处的按钮 + 符号，就会出现中间的弹出窗口，在该窗口中 3 号 位置选择图片中的 biosboot （不要手动输入界面的问题，请使用既有的选项来选择），同时输入大约 2M 的容量，按下 添加挂载点 后，就会显示该分区的详细数据。

如上图所示，界面的右边就是 biosboot 分区的详细数据。由于是 BIOS 使用，因此没有挂载点，同时文件系统的部分也是会变成 BIOS Boot 的关键词，并不会是Linux的文件系统，接下来，要来设置其他的分区。继续点击下面的 + 号。

另外，图中的 设备类型 其实有3种，这里只使用了标准分区和LVM。三种设备类型的意义分别如下：

  • 标准分区：类似 /dev/vda1 之类的分区。
  • LVM：这是一种可以弹性增加或减小文件系统容量的分区，之后或许会在我总结的 《鸟哥的linux私房菜 》中说明。
  • LVM精简配置（Thin Provisioning）：这个是LVM的高级版。与传统LVM直接分配固定的容量不同，这个 LVM精简配置 的选项，可以在使用多少容量才分配磁盘多少容量给你，所以如果LVM设备内的数据量较少，那么你的磁盘其实还可以做更多的数据存储，而不会被平白无故的占有。

另外，图中的文件系统就是实际 格式化 的时候，我们可以格式化成什么文件系统的意思。下面简略分别说下各个文件系统的选项：

  • ext2/ext3/ext4：这是 Linux 早期使用的文件系统类型。由于 ext3/ext4 文件系统多了日志功能，对于系统的恢复比较快速，不过由于磁盘容量越来越大，ext 系列似乎有点挡不住了，所以除非有特殊设置需求，否则进来比较少使用 ext4。
  • swap：就是磁盘模拟为内存的交换分区，由于交换分区并不会使用到目录树的挂载，所以用交换分区就不需要指定挂载点。
  • BIOS Boot：就是 GPT 分区表可能会使用到的东西，若使用的是 MBR，就不需要这个东西了。
  • xfs：这个是目前 CentOS 7 默认的文件系统，最早是为大型服务器所开发。它对于大容量的磁盘管理非常好，而且格式化的时候速度相当快，很适合当今动不动就是好几个 TB 的磁盘的环境，因此主要使用这个。
  • vfat：同时被 Linux 与 Windows 所支持的文件系统类型。如果主机硬盘内同时存在 Windows与Linux系统，为了数据交换，确实可以创建一个 vfat 的文件系统。

接下来是建立 /boot 挂载点的文件系统，容量部分可以输入 1G 或是 1024M 都可以，然后按下 新增 ，就会回到同一配置页面。

接下来依序建立另外所需要的根目录 / 分区，建议给与 10GB 的容量，我们的 / 、 /home 、 交换分区都使用LVM磁盘管理模式即可。

选择分区 / 然后点击设备类型选择为 LVM ， 由于 LVM 会取一个默认名字的 LVM 设备，如果需要修改，按下 修改 按钮即可，接下来会出现下面的界面，处理 LVM 的相关设置

这里大小根据需求选择即可，在大小策略部分选择 固定容量，大小根据需求设置。

之后创建 /home 挂载点，同样设置为 LVM 类型，之后确定卷组部分还有剩余容量即可。

然后设置 swap 分区（交换分区）是当物理内存容量不够时，可以拿这部分来存放内存中较少被使用的数据，以前都建议交换分区需要时内存的2倍最佳。现在的内存都比较大，但是还是最好保持有，不过也不需要太大，1——2GB就好。同样设置为 LVM 类型，之后确定卷组部分还有剩余容量即可。

内存交换分区的功能是：当有数据被存放在物理内存里面，但是这些数据又不是长被CPU所使用时，那么这些不常被使用的数据将会被扔到硬盘的交换分区当中，而将速度较快的物理内存空间释放出来给真正需要的程序使用，所以，如果你的系统不很忙，而内存又很大，自然不需要交换分区。

设置好密码后，按下两次 完成，安装程序才会真的帮你设置该密码。

创建用户

接下来建立一个日常登录系统的常用账号，建议将管理员直接登录的权限拿掉，有需要时才用特殊命令（如 su，sudo等命令）切换成管理员身份，所以这里创建一个一般账号

开始安装

点击 开始安装 按钮，进行安装。

安装完成之后，点击 重启系统 按钮

准备使用系统前的授权同意

重新启动完毕后，就会进入下图的页面，等待登录，就此安装完成。（其他系统可能会进入第一次使用的许可证接受界面，选择同意协议后，再次确认后，就可以进入登录页面）

其他问题

桥接模式下无法联网问题

桥接模式下，是通过网络适配器 VMware Network Adapter VMnet0 来连接网络，可以先打开 控制面板 -> 网络和Internet -> 网络连接 中来查看网络适配器设置。

如果没有找到 VMnet0 的网络适配器就需要手动添加下。

打开 VMware 编辑 -> 虚拟网络编辑器 来配置网络

点击更改设置来添加网络适配器。点击后会提示需要管理员权限启动，确认下VMnet0的配置

打开虚拟机的 配置 -> 网络适配器 -> 将桥接模式下的复制物理网络连接状态勾选。

查看宿主机的网络连接配置，即 默认网关、DNS服务器

  1. 通过设置 -> 网络和 Internet -> 状态 -> 查看硬件和连接属性， 找到连接性是已连接的配置即可查看

  2. 通过设置 -> 网络和 Internet -> 状态 -> 更改适配器选项，双击接入了网络的适配器，点击详情信息查看配置

打开虚拟机，进入网络配置页面

点击 设置

选择 IPv4 选项，将其中的子网掩码、网关、DNS配置与宿主机一致（DNS可以只用配置一个）

保存之后，测试网络连接
